# Longshot v2

Daily 4×4 word puzzle. Three shots. Find the longest word.

## Structure

```
apps/longshot/
  index.html         — game shell (HTML/head only, no logic)
  styles.css         — all visual styles (dark theme)
  main.js            — app entry point, wires everything together
  game.js            — core game logic (path selection, submission, medals)
  ui.js              — DOM rendering and interaction
  board-bank.js      — loads board-bank.json, selects today's board
  storage.js         — localStorage: daily state + streak
  share.js           — spoiler-free result sharing
  help.js            — help modal
  data/
    longshot-common-words.txt  — filtered gameplay lexicon (generated)
    longshot-allowlist.txt     — words to force-include in lexicon
    longshot-denylist.txt      — words to exclude from lexicon
    board-bank.json            — precomputed boards (generated by build script)
    vendor/
      wordfreq-en-25000-log.json  — vendored upstream lexicon source
      README.md                   — upstream attribution / licence notes
  README.md

scripts/
  build-longshot-lexicon.mjs     — generates longshot-common-words.txt
  build-longshot-board-bank.mjs  — generates board-bank.json from seed grids
  inspect-longshot-board.mjs     — diagnostic: test words against boards
```

## Lexicon source & attribution

The gameplay word list is derived from **wordfreq-en-25000**, published by
[Allison Parrish](https://github.com/aparrish/wordfreq-en-25000) and based on
the [wordfreq](https://github.com/rspeer/wordfreq) library by Robyn Speer /
Luminoso Technologies (MIT licence).

The upstream file (`wordfreq-en-25000-log.json`) is vendored at
`apps/longshot/data/vendor/` for reproducible builds. See
`apps/longshot/data/vendor/README.md` for full attribution and refresh
instructions.

**No raw wordfreq data reaches the browser.** Only the per-board precomputed
word sets inside `board-bank.json` are served to players.

## Lexicon pipeline

Run in order after changing the allowlist, denylist, or upstream source:

```bash
# Step 1 — rebuild filtered gameplay lexicon from vendored upstream
node scripts/build-longshot-lexicon.mjs

# Step 2 — rebuild per-board word sets from updated lexicon
node scripts/build-longshot-board-bank.mjs
```

### Allowlist / denylist

| File | Purpose |
|------|---------|
| `longshot-allowlist.txt` | Force-include specific words regardless of rank |
| `longshot-denylist.txt`  | Exclude specific words even if they're in the top N |

Both files support `# comments`. One word per line, lowercase, 5+ letters.

## How validation works

The game never does runtime path-finding. At build time, `build-longshot-board-bank.mjs` runs DFS on each board and writes every valid traceable 5+ letter word into `board-bank.json`. At runtime, `game.js` checks if the submitted word is in `board.allowed` (binary search on a sorted array).

This makes validation:
- **Fast** — O(log n) per check, no graph traversal at runtime
- **Consistent** — all players see the same allowed word set
- **Transparent** — the data file is in the repo, auditable
- **Trust-first** — the lexicon covers ~15,000 common English words from a
  verified frequency-ranked source, so ordinary words are not spuriously rejected

## Diagnostics

```bash
# List all boards
node scripts/inspect-longshot-board.mjs

# Show a specific board's grid and word list
node scripts/inspect-longshot-board.mjs --board 5

# Test a word on all boards
node scripts/inspect-longshot-board.mjs --word rating

# Test a word on a specific board (shows DFS path)
node scripts/inspect-longshot-board.mjs --board 5 --word rating

# Look up which board is used for a given date (YYYY-MM-DD)
node scripts/inspect-longshot-board.mjs --date 2026-03-21
```

## Board bank

Board selection is deterministic by local calendar date: the board index is derived by hashing the date string (YYYY-MM-DD) modulo the bank size. Same date → same board for all players.

Each board has:
- `grid` — 16-letter array (4×4, row-major)
- `allowed` — sorted array of all traceable 5+ letter words from the gameplay lexicon
- `featured` — the "Longshot" word (longest word on the board)
- `topWords` — top 10 words by length (shown on results screen)
- `medals` — `{ bronze: 5, silver: 6, gold: 7 }`

## Expanding the board bank

Add new 16-char grid strings to `SEED_GRIDS` in `build-longshot-board-bank.mjs`,
then run `node scripts/build-longshot-board-bank.mjs`. Preferred boards have a
7+ letter featured word and at least one 6-letter word.

## Local development

No build step required — all static files. Just serve from repo root:

```bash
npx serve .
# visit http://localhost:3000/apps/longshot/
```

## Deployment
